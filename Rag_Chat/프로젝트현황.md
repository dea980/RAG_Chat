# 프로젝트 현황 (2026-02-03)
Palantir 제출용으로 현재 동작/이슈/다음 액션을 요약합니다.

## 동작 중인 것
- Streamlit 기반 챗 UI → Django API → 벡터 검색(FAISS/Chroma) → Gemini/Qwen LLM 응답
- 질문·컨텍스트·응답을 `Chat`, `SearchLog`에 기록
- Redis 세션 유지, 활동 갱신 API 존재
- Provider 프리셋 API + Streamlit 토글 적용 가능
- run_local_fixed.sh / docker-compose로 로컬 일괄 기동

## 주요 이슈
- 인증·RBAC·금지어 필터 없음 (엔드포인트 공개 전제)
- Redis TTL과 DB `expired_datetime` 정합성 부족 → 세션 만료 재현 어려움
- 스트리밍 미지원, 응답 지연 시 UX 저하
- 모니터링·헬스체크·APM 없음
- 다중 노드/오토스케일 설계 미구현

## 단기 개선 계획
1) 세션 만료 정합성: Redis TTL과 DB 필드 동기화, 만료시 로그 남기기  
2) 기본 헬스 체크: LLM/Redis/DB 핑 + 실패 시 경고 로그  
3) 금지어/역할 필터: Django 미들웨어 훅을 추가하고 기본 룰셋 적용  
4) 최소 모니터링: Celery/LLM 호출 지연 메트릭을 로깅으로라도 수집  
5) 리랭커 옵션 검토: bge-reranker 등 플러그인화, 품질 변화 측정
